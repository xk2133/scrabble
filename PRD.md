# 英语棋（English Word Chess）产品需求文档

---

## 1. 产品概述

### 1.1 产品名称
**英语棋（English Word Chess）** — Scrabble 类网页版单词对战游戏

### 1.2 产品定位
一款融合游戏化对战与英语学习的 H5 网页版 Scrabble 游戏。让用户在经典拼字博弈中自然提升词汇量，实现"游戏即学习"的无缝体验。

### 1.3 目标用户

| 用户群体 | 年龄段 | 特征描述 |
|---------|--------|---------|
| 英语学习者（青少年） | 10-18 岁 | 以游戏化方式背单词，需要趣味性和学习辅助 |
| 英语学习者（成人） | 20-40 岁 | 碎片时间保持词汇量，需要便捷性和竞技性 |
| 社交游戏玩家 | 全年龄段 | 微信群分享链接与朋友对战，需要社交传播能力 |
| Scrabble 爱好者 | 全年龄段 | 寻找纯净的在线 Scrabble 体验，反感广告和付费制胜 |

### 1.4 核心问题

**用户侧痛点：**
- 现有背单词工具（百词斩、墨墨等）枯燥重复，缺乏真实应用场景，学习留存率低
- 英语单词学习与使用场景脱节，用户记忆了大量单词却缺乏拼写实践

**市场侧空白：**
- Scrabble GO 等国际产品广告泛滥（每局强制观看 30 秒视频广告）、道具付费制胜、无学习功能
- 中国市场 Scrabble 网页版完全空白，中文用户缺少本土化的英语拼字对战产品
- 现有英语学习产品与游戏场景割裂，用户需要切换多个 App

### 1.5 产品愿景
打造中国最佳的英语拼字对战平台，让每一位英语学习者在博弈对抗中自然提升词汇量，让拼字游戏成为连接学习与娱乐的桥梁。

## 2. 市场分析

### 2.1 市场现状

| 维度 | 现状分析 |
|------|---------|
| 全球 Scrabble 市场 | Scrabble GO 全球月活约 500 万，但用户满意度低（App Store 3.8 分），主要投诉为广告和付费机制 |
| 中国市场 | Scrabble 品类在中国几乎空白，无本土化成熟的网页版产品，市场蓝海 |
| 英语学习市场 | 中国在线英语教育市场规模超 500 亿元，单词记忆类 App 用户超 1 亿 |
| 小游戏生态 | 微信小游戏日活用户超 4 亿，H5 链接分享是低成本获客的核心渠道 |

### 2.2 竞品分析

| 竞品 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|-------------|
| Scrabble GO | 品牌知名度高、用户量大 | 广告泛滥、道具付费制胜、无学习功能、无中文支持 | 无广告、学习引擎、中文 UI、微信适配 |
| Words With Friends | 社交体验好 | 异步对战为主、对中文用户不友好 | 实时对战、单词释义发音 |
| 百词斩 | 用户量大、词库丰富 | 纯学习工具、无对战竞技 | 游戏化对战、拼写实践 |
| Wordscapes | 玩法新颖 | 单人游戏、无真人对战 | 多人实时对战、社交传播 |

### 2.3 用户痛点

1. **广告打断体验**：Scrabble GO 每局强制广告，严重破坏游戏沉浸感
2. **付费破坏公平**：道具付费模式导致付费玩家拥有显著优势
3. **学习场景缺失**：国际 Scrabble 产品不提供单词释义、发音等学习功能
4. **中文适配缺失**：无中文界面、无法在微信生态内流畅分享和运行
5. **拼写实践不足**：传统背单词工具缺少拼写输出环节，学习效果打折扣

### 2.4 市场机会

1. **品类空白红利**：中国市场 Scrabble 网页版完全空白，先发优势明显
2. **学习+游戏双需求**：同时切入英语学习和休闲游戏两个巨大市场
3. **社交裂变潜力**：微信生态 H5 链接分享天然具备社交裂变传播基础
4. **用户留存天然高**：对战竞技天然驱动用户反复使用，留存率优于纯学习工具

### 2.5 商业价值（远期展望）

- **基础服务免费**：核心对战 + 学习功能永久免费，获取最大用户基数
- **增值服务探索**：高级 AI 难度、自定义棋盘主题、高级词库包（非 MVP 阶段）
- **教育合作**：与英语培训机构合作提供定制化教学版本

### 3.2 用户旅程

**核心用户旅程（学生小明首次使用 → 对战获胜 → 学习收藏）：**

```
发现阶段：微信群收到好友分享的对战链接
  ↓
进入阶段：点击链接直接进入游戏大厅（无需注册/下载）
  ↓
探索阶段：查看大厅界面，看到"快速匹配"和"AI 对战"选项
  ↓
首次体验：选择"AI 对战（初级）"，进入 15×15 棋盘
  ↓
对战中：首次放置单词 "CAT"，系统自动计分，弹出单词卡片（中文释义 + 发音）
  ↓
学习时刻：点击发音按钮听 TTS 发音，点击"收藏"加入单词本
  ↓
棋局推进：AI 回应放置单词，用户继续思考下一个单词
  ↓
结束体验：对局结束，显示比分和本局学习的单词列表
  ↓
留存传播：生成对战结果分享卡片，发送回微信群
  ↓
二次访问：打开单词本复习收藏的单词，再次发起对战
```

### 4.3 页面详细线框图描述

#### 4.3.1 首页/大厅页 (P01)

**布局结构**：垂直居中单列布局，移动端全屏适配

**关键元素（从上到下）：**
- **顶部品牌区**：居中显示游戏 Logo（字母拼块图标）和标题"英语棋 · English Word Chess"，下方一行副标题"拼出你的词汇力"
- **快速匹配区**：大号主按钮"⚡ 快速匹配"，橙色/暖色系高亮，点击后自动匹配在线真人玩家，匹配中显示加载动画和预计等待时间
- **AI 对战区**：按钮"🤖 与 AI 对战"，点击后展开难度选择面板（初级·中级），选择后直接创建房间开始对战
- **房间操作区**：
  - "创建房间"按钮，点击后生成房间号并自动复制到剪贴板，提示"房间号已复制，发送给好友即可对战"
  - 输入框 + "加入房间"按钮，用户粘贴房间号后点击加入
- **底部导航栏**（固定在视口底部）：
  - 三个 Tab 图标 + 文字：大厅（首页图标，当前高亮）、单词本（书本图标）、帮助（问号图标）
  - 高度 56px，底部安全区适配

**响应式设计**：
- 移动端（< 768px）：单列布局，按钮全宽，内边距 16px
- 平板/桌面（≥ 768px）：内容区最大宽度 480px 居中，两侧留白

#### 4.3.2 游戏对战页 (P02)

**布局结构**：从上到下依次为顶栏、棋盘区、手牌区，全屏沉浸式

**关键元素（从上到下）：**
- **顶部信息栏**（高度 48px）：
  - 左侧：对手头像（首字母圆形头像）+ 对手昵称 + 对手当前得分
  - 中间：回合倒计时（环形进度条 + 秒数），默认 60 秒
  - 右侧：己方昵称 + 当前得分 + 己方头像
  - 得分字体加大加粗，获得分数时播放数字跳动动画
- **棋盘区域**（占据主要视觉空间，Canvas 渲染）：
  - 15×15 正方形网格，每个格子为正方形
  - 特殊格子颜色标识：
    - 三倍单词分（TW）：红色
    - 双倍单词分（DW）：粉色
    - 三倍字母分（TL）：蓝色
    - 双倍字母分（DL）：浅蓝色
    - 中心星标格（★）：灰色/金色
  - 已放置的字母块：米色/木质纹理方块，字母居中，右下角小字显示字母分值
  - 当前回合预览放置：半透明方块，区别于已确认的字母
  - 棋盘支持触摸拖动平移和双指缩放（移动端）或鼠标滚轮缩放（桌面端）
- **手牌区域**（固定在视口底部）：
  - 7 个字母方块横向排列，每个方块尺寸适配屏幕（移动端约 42px × 42px）
  - 字母方块可拖拽到棋盘或点击选中后再点击棋盘放置
  - 已选中的字母方块有高亮边框动画
  - 字母方块上显示字母和右下角分值
- **操作按钮栏**（手牌下方）：
  - "提交"按钮（绿色）：确认当前放置的单词，无效单词时置灰不可点击
  - "跳过"按钮（灰色）：放弃当前回合，确认弹窗二次确认
  - "交换"按钮（灰色）：用选中的字母交换新的随机字母，消耗回合
  - 按钮排列方式：提交占主要位置（宽度 60%），跳过和交换各占 20%
- **消息提示区域**（棋盘上方或 Toast）：
  - 系统提示："轮到你了"、"对方已提交单词 【单词名】+"分数"分"
  - 错误提示："【单词】不在词库中"、"字母未连接"
  - 奖励提示："🎉 BINGO! 额外 +50 分"
- **游戏状态标记**：棋盘旁显示当前回合玩家标识（高亮边框或指示箭头）

**响应式设计**：
- 移动端（< 768px）：棋盘占视口宽度的 95%，格子边长自动计算（≈ (viewportWidth - 32) / 15），手牌方块 42px
- 平板（768px - 1024px）：棋盘最大 600px，居中布局
- 桌面（> 1024px）：棋盘 600px，手牌方块 52px

#### 4.3.3 单词详情弹窗 (P03)

**布局结构**：居中弹出式卡片浮层，半透明黑色遮罩背景

**关键元素（从上到下）：**
- **遮罩层**：半透明黑色背景（opacity: 0.5），点击遮罩关闭弹窗
- **卡片容器**（白色圆角卡片，移动端宽 85%，最大 360px）：
  - **顶部关闭区**：右上角 X 关闭按钮
  - **单词展示区**：单词大号字体居中显示，下方标注音标
  - **发音按钮**：喇叭图标 + "发音"文字按钮，点击调用 TTS 播放，播放中显示声波动画
  - **释义区**：标签"中文释义"，下方显示中文翻译（支持多个释义，用逗号分隔）
  - **例句区**（如有数据）：标签"例句"，显示英文例句和中文翻译
  - **词性标签**：显示词性（n./v./adj. 等）
  - **底部操作**：
    - "⭐ 收藏"按钮：切换收藏状态，已收藏显示"★ 已收藏"（黄色填充）
    - "了解详情"链接（可选）：跳转外部词典页面
- **加载状态**：弹窗打开时显示骨架屏加载，数据返回后渐显内容
- **异常状态**：单词未找到释义时显示"暂无释义数据"

**触发方式**：
- 棋盘上长按/右键点击已放置的字母块
- 对手提交单词后系统消息中的单词链接
- 对局结束结算面板中的单词列表

**响应式设计**：
- 移动端：卡片从底部滑入（类似 iOS Action Sheet），高度自适应内容（最大 60vh）
- 桌面端：居中弹出式 Modal，固定宽度 360px

#### 4.3.4 单词本页 (P04)

**布局结构**：顶部导航 + 单词列表，垂直滚动

**关键元素（从上到下）：**
- **顶部导航栏**：左侧返回箭头，中间标题"我的单词本"，右侧可选搜索图标
- **统计横幅**（可选，列表顶部）：累计收藏单词数、本周新增数
- **单词列表**：
  - 每个单词卡片包含：单词（左侧，加粗）、中文释义（右侧）、发音按钮（喇叭图标）
  - 卡片右滑显示"取消收藏"操作（iOS 风格滑动删除）
  - 点击单词卡片弹出单词详情弹窗
  - 列表按收藏时间倒序排列
  - 支持下拉刷新
- **空状态**：未收藏任何单词时，居中显示书本图标 + "还没有收藏单词，去对战中发现新单词吧！"+ "去对战"按钮（跳转首页）
- **复习模式入口**（可选悬浮按钮）：点击进入单词卡片复习模式（正面英文、背面中文释义）

**响应式设计**：
- 移动端：单列列表，卡片全宽
- 桌面端：列表最大宽度 600px 居中

## 5. 功能需求

### 5.1 功能分级总览

| 优先级 | 定义 | MVP 是否包含 |
|-------|------|-------------|
| P0 | 核心必备功能，缺少则产品无法运行 | ✅ 是 |
| P1 | 重要功能，提升核心体验完整性 | ✅ 是 |
| P2 | 锦上添花，未来版本迭代 | ❌ 否 |

### 5.2 P0 功能需求表

| 编号 | 功能名称 | 功能描述 | 用户价值 | 业务规则 | 输入 | 输出 | 异常处理 | 技术要求 |
|------|---------|---------|---------|---------|------|------|---------|---------|
| F-P0-01 | 15×15 Scrabble 棋盘渲染 | 使用 Canvas 渲染标准 Scrabble 15×15 棋盘，含特殊格子标识和字母放置 | 提供核心对战画布 | 棋盘尺寸 15×15，中心星标格位于 (7, 7)，特殊格子布局遵循标准 Scrabble 规则（TW 8个、DW 17个、TL 12个、DL 24个） | 无 | Canvas 渲染的棋盘画面 | Canvas 加载失败时降级为 DOM 网格渲染 | React + Canvas API，棋盘状态用二维数组管理 |

## 6. 用户故事

### 6.1 P0 功能用户故事

#### US-P0-01：经典 Scrabble 对战
**作为** 一名 Scrabble 爱好者
**我想要** 在手机上与其他玩家进行标准的 15×15 Scrabble 对战
**以便** 享受纯粹的拼字博弈乐趣

**验收标准：**
- [ ] 棋盘为 15×15 标准 Scrabble 布局，特殊格子颜色和位置正确
- [ ] 先手玩家首回合必须覆盖中心星标格
- [ ] 每次放置后自动计算得分，单词提交后计分立即展示
- [ ] 得分动画在 1 秒内完成
- [ ] 回合倒计时 60 秒，剩余 10 秒时倒计时变红并闪烁
- [ ] 超时自动跳过，手牌不变
- [ ] Bingo（7 字母全部用完）正确额外奖励 50 分

**业务规则：**
- 字母分值和分布遵循标准 Scrabble 规则（100 个字母块的标准分布）
- 每回合必须使用手牌中至少 1 个字母
- 新放置字母必须与棋盘已有字母形成完整合法单词
- 交换字母消耗一个回合，不获得分数
- 连续 3 次双方均跳过时对局结束

**前置条件：** 两名玩家已进入同一房间并准备就绪
**后置条件：** 对局结束，展示结算面板


#### US-P0-02：AI 对战
**作为** 一名英语初学者
**我想要** 与不同难度的 AI 进行 Scrabble 对战
**以便** 在没有真人对手时也能练习和娱乐

**验收标准：**
- [ ] 大厅页点击"与 AI 对战"弹出难度选择面板
- [ ] 初级 AI 在 500ms 内完成响应
- [ ] 中级 AI 在 2s 内完成响应
- [ ] AI 下的单词始终是合法单词
- [ ] AI 对局与真人对局使用相同的棋盘和计分规则
- [ ] 对局结束后正常展示结算面板

**业务规则：**
- 初级 AI：贪心算法，选择当前最高分单词
- 中级 AI：Minimax 算法，考虑 2-3 步前瞻
- AI 响应过程中显示"对手思考中..."状态

**前置条件：** 用户在大厅页选择 AI 对战
**后置条件：** 正常完成对局并展示结果


#### US-P0-03：单词学习辅助
**作为** 一名英语学习者
**我想要** 在对战中随时查看单词的中文释义和发音
**以便** 在游戏过程中自然学习新单词

**验收标准：**
- [ ] 点击棋盘上任意已放置的单词，弹出单词详情卡片

- [ ] 网络异常时使用缓存数据显示
- [ ] 弹窗可通过点击遮罩或关闭按钮关闭

**业务规则：**
- 释义数据优先使用本地缓存，缓存 TTL 为 7 天
- Web Speech API 不可用时隐藏发音按钮
- 发音语言固定为 en-US

**前置条件：** 棋盘上有已确认放置的单词
**后置条件：** 用户关闭弹窗返回棋盘

## 7. 用户流程

### 7.1 主要操作路径

#### 7.1.1 路径一：AI 对战（首次用户体验主路径）

```
用户打开链接
  → 进入大厅页（P01）
  → 点击"🤖 与 AI 对战"
  → 弹出难度选择面板：选择"初级"或"中级"
  → 进入游戏对战页（P02）
  → 系统决定先手（随机）
  → 开始对战循环：
      ├── 己方回合：
      │   ├── 从手牌拖拽字母到棋盘
      │   ├── 点击"提交"（校验单词合法性）
      │   ├── 合法 → 单词放置确认 + 得分动画 → 轮到 AI
      │   └── 非法 → 红色提示 + 字母退回手牌
      ├── AI 回合：
      │   ├── 显示"对手思考中..."
      │   ├── AI 放置单词
      │   └── 棋盘更新 + 消息提示"对手下了 【单词】+分数" → 轮到用户
      └── 单词学习交互：
          └── 用户长按任意已放置单词 → 弹出单词详情弹窗（P03）
              ├── 查看释义
              ├── 点击发音
              └── 收藏到单词本
  → 字母袋耗尽且一方手牌用完 → 对局结束
  → 弹出结算面板（P05）
  → 用户选择"再来一局"或返回大厅
```

#### 7.1.2 路径二：好友对战

```
用户在大厅页
  → 点击"创建房间"
  → 获得 6 位房间号，自动复制
  → 分享给好友（微信/复制链接）
  → 进入房间等待页（显示"等待对手加入..."）
  → 好友通过链接/房间号加入
  → 双方就位，自动开始对战
  → 对战循环（同路径一）
  → 对局结束 → 结算面板 → 分享战绩
```

#### 7.1.3 路径三：加入好友房间

```
用户收到好友分享的链接或房间号
  → 点击链接（自动填充房间号并跳转大厅）
  → 或在大厅页输入房间号点击"加入"
  → 系统校验房间号
      ├── 有效且未满 → 进入房间
      └── 无效 → 提示"房间不存在" → 返回大厅
  → 双方就位，开始对战
```

#### 7.1.4 路径四：单词本复习

```
用户在大厅页点击底部导航"单词本"
  → 进入单词本页（P04）
  → 查看收藏的单词列表
  → 点击单词发音
  → 点击单词卡片查看详情（P03）
  → 管理收藏（滑动删除）
  → 返回大厅继续对战
```

```
              └──────────┘    └──────────┘
```

### 7.3 异常流程处理

| 异常场景 | 处理流程 | 用户提示 |
|---------|---------|---------|
| WebSocket 连接断开 | 自动重连（最多 5 次，间隔 2s/4s/8s/16s/32s 递增），重连成功后恢复房间状态 | "连接断开，正在重连...（第 N 次）" |
| 断线超过 30 秒 | 判定为弃权，对手获胜 | "对手已断开连接，你赢了！" |
| 单词不在词库中 | 拒绝提交，字母退回手牌，棋盘预览清除 | "【单词】不在词库中，请检查拼写" |
| 字母未与棋盘连接 | 拒绝提交 | "请将字母与棋盘上已有的字母相连" |
| Free Dictionary API 不可用 | 使用 LocalStorage 缓存数据，缓存未命中时显示占位信息 | "释义暂不可用，请稍后再试" |
| Web Speech API 不可用 | 隐藏发音按钮，不显示相关 UI | 无提示（静默降级） |
| 快速匹配超时（30s） | 提示并提供 AI 对战备选 | "暂无在线玩家，试试 AI 对战吧！" |
| 房间不存在或已满 | 提示具体原因 | "房间不存在" 或 "房间已满员" |
| LocalStorage 不可用 | 单词本功能降级，提示用户 | "单词本功能暂不可用（浏览器存储受限）" |
| 字母袋耗尽 | 正常结算对局 | 正常进入结算流程 |
| AI 计算超时（>3s） | AI 自动跳过回合 | "对手跳过回合" |
| 棋盘 Canvas 渲染失败 | 降级为 DOM 网格渲染 | 无感知降级 |

## 8. 产品约束

### 8.1 平台要求

| 约束项 | 要求 |
|-------|------|
| 运行环境 | H5 网页版，移动端优先 |
| 目标浏览器 | Chrome 90+、Safari 14+、微信内置浏览器（X5/WebKit）、Edge 90+ |
| 最低屏幕宽度 | 320px（iPhone SE 等小屏设备） |
| 推荐屏幕宽度 | 375px - 428px（主流手机） |
| 网络环境 | 支持 3G/4G/5G/WiFi，弱网环境需有降级策略 |

### 8.2 性能要求

| 指标 | 目标值 | 测量方式 |
|------|-------|---------|
| 首屏加载时间（FCP） | < 1.5s（4G 网络） | Lighthouse |
| 可交互时间（TTI） | < 3s | Lighthouse |
| AI 响应时间（初级） | < 500ms | 服务端计时 |
| AI 响应时间（中级） | < 2s | 服务端计时 |
| WebSocket 消息延迟 | < 200ms | 端到端计时 |
| 单词校验时间 | < 100ms | 客户端 Trie 树查询 |
| 页面切换动画 | 60fps | Chrome DevTools FPS |
| 棋盘触摸响应 | < 50ms | 触摸事件延迟 |
| 内存占用 | < 150MB（移动端） | Chrome DevTools Memory |

### 8.3 功能边界

| 边界类型 | MVP 范围 | 明确不做 |
|---------|---------|---------|
| 对战人数 | 2 人 | 3-4 人对战 |
| AI 难度 | 初级 + 中级 | 高级 AI（Monte Carlo） |
| 用户体系 | 无注册，匿名使用 | 注册登录、账号体系 |
| 排行榜 | 不做 | 全服排行、好友排行 |
| 棋盘模式 | 标准 Scrabble 15×15 | 自定义棋盘尺寸 |
| 词库范围 | 标准英文词库（ENABLE/CSW） | 自定义词库导入 |
| 社交功能 | 链接分享 + 房间号 | 好友系统、聊天 |
| 支付 | 无 | 任何付费功能 |

### 8.4 内容规范

| 约束项 | 规范 |
|-------|------|
| UI 语言 | 简体中文 |
| 游戏内容语言 | 英文（单词本身） |
| 发音标准 | 美式英语（en-US） |
| 词库标准 | ENABLE（北美）或 CSW（国际），MVP 使用 ENABLE |
| 违规词过滤 | 词库预过滤种族歧视、色情等不当词汇 |

### 8.5 技术约束

| 约束项 | 要求 |
|-------|------|
| 前端框架 | React 18+ + TypeScript |
| 构建工具 | Vite 5+ |
| 棋盘渲染 | Canvas API（主要）+ DOM 降级（备用） |
| 状态管理 | React Context + useReducer（不引入 Redux 等重型库） |
| 路由 | React Router v6 |
| WebSocket | Socket.io v4 |
| 后端语言 | Node.js（推荐）或 Python FastAPI |
| 词库数据结构 | Trie 树（前端）+ 哈希表/数据库（后端） |
| 数据持久化 | LocalStorage（前端）+ 内存存储（后端，MVP 不引入数据库） |
| CDN | 静态资源部署到 CDN |

### 8.6 合规要求

| 约束项 | 要求 |
|-------|------|
| 数据隐私 | MVP 不收集用户个人信息，无需注册，符合 GDPR/《个人信息保护法》最小化原则 |
| 内容审核 | 词库预过滤不当内容，玩家不可自定义输入非单词文本 |
| 儿童保护 | 无聊天功能，无社交私信，避免未成年人社交风险 |
| 版权合规 | Scrabble 商标属于 Hasbro，产品名使用"英语棋"而非"Scrabble"；词库使用开源 ENABLE 词库 |
| Cookie 声明 | 使用 LocalStorage 存储单词本数据，首次使用时简洁提示 |

### 8.7 兼容性要求

| 平台 | 要求 |
|------|------|
| iOS Safari | iOS 14+，支持 WebSocket、Web Speech API、Clipboard API |
| Android Chrome | Chrome 90+，全功能支持 |
| 微信内置浏览器（iOS） | WebKit 内核，需测试 WebSocket 和 Canvas 兼容性 |
| 微信内置浏览器（Android） | X5 内核，需测试 Touch 事件和分享 API |
| 桌面 Chrome | 响应式布局适配，棋盘最大 600px |
| 桌面 Safari | 响应式布局适配 |
| PWA 支持 | MVP 不做离线 PWA，但保留 manifest.json 为未来做准备 |
