## 1. 设计概述

### 1.1 设计理念：「阳光书房」

"阳光书房"是一个将温暖、学习、游戏三种体验融为一体的设计语言体系。设计灵感源自午后洒满阳光的书房角落——温暖的木质家具、柔和的自然光线、书架上排列的书籍，以及一只聪明又友好的猫头鹰伙伴 Ollie。

**核心隐喻**：用户在"书房"中通过拼字游戏"翻阅"英语词汇，阳光（高亮色）引导操作路径，木质（暖棕底色）提供安定感，Ollie（猫头鹰吉祥物）作为学习伙伴全程陪伴。

### 1.2 品牌定位

- **品牌名称**：英语棋 · English Word Chess
- **品牌口号**：拼出你的词汇力
- **品牌人格**：温暖、聪明、友善、可靠——像一个耐心的英语老师，又像一个有趣的游戏伙伴
- **吉祥物**：Ollie（猫头鹰），圆润可爱、大眼明亮，象征智慧和洞察力
- **差异化定位**：中国市场唯一定位"学习+游戏"双场景的 Scrabble 网页版，无广告、暖设计、中文友好

### 1.3 目标用户

| 用户群体 | 设计关注点 |
|---------|-----------|
| 青少年英语学习者（10-18岁） | 操作直观、反馈丰富、学习功能醒目、Ollie引导亲切 |
| 成人英语学习者（20-40岁） | 界面清爽不幼稚、信息密度适中、碎片时间友好 |
| 社交游戏玩家 | 分享路径短、对战入口显著、视觉有品质感 |
| Scrabble 爱好者 | 棋盘专业感、规则完整、无干扰纯净体验 |

### 1.4 设计原则

1. **一目了然**（At-a-Glance）：用户进入任何页面，3秒内理解界面用途和核心操作入口。信息层级清晰，视觉动线自上而下。
2. **点哪哪响**（Multi-Sensory Feedback）：每次操作都有即时、明确、多感官（视觉+听觉+触觉）的反馈——按钮按下的弹性变形、字母牌放置的清脆音效、得分跳动的数字动画。
3. **先表扬再纠正**（Encourage Before Correct）：当用户犯错（如提交无效单词），先给予正向引导（"差一点！这个词拼写有点问题"），再提供具体建议，避免挫败感。Ollie 表情随情景变化（开心/思考/鼓励）。
4. **渐进成长**（Progressive Growth）：界面复杂度随用户熟练度自然展开。首次使用时 Ollie 引导教学局，熟练后仅保留必要信息（倒计时、分数），减少干扰。
5. **暖而不燥**（Warm but Not Noisy）：温暖的色彩系统以低饱和暖色为主，高亮色（阳光黄#FFD23F）仅在关键交互点使用，避免大面积高饱和色造成的视觉疲劳。
6. **无感性能**（Invisible Performance）：加载状态用可爱的 Ollie 动画替代冰冷的进度条；弱网时优雅降级，不打断用户心流。

### 1.5 设计目标

- 3秒可理解：任何页面核心操作入口在3秒内被用户发现
- 操作零困惑：所有可交互元素有清晰的视觉可供性（affordance）
- 学习无形：单词释义、发音等学习功能贯穿游戏流程，但不打断对战节奏
- 品牌记忆：猫头鹰Ollie和"阳光书房"暖色调形成强品牌识别
- 全设备一致：320px小屏到桌面大屏的主操作体验一致

#### 2.3.4 圆角规范（Claymorphism 风格）

Claymorphism（黏土形态）风格是本设计的核心视觉特征——元素具有柔软、圆润、立体的外观，像黏土手工制作一样有亲和力。

```css
:root {
  /* 圆角系统 */
  --radius-xs: 6px;    /* 极小圆角：字母牌、标签、徽章 */
  --radius-sm: 10px;   /* 小圆角：输入框、小按钮、Toast */
  --radius-md: 14px;   /* 中圆角：卡片、面板、大按钮 */
  --radius-lg: 20px;   /* 大圆角：弹窗、模态框、棋盘 */
  --radius-xl: 28px;   /* 超大圆角：Ollie 头像、圆形按钮 */
  --radius-full: 50%;  /* 圆形：头像、图标按钮 */

  /* Claymorphism 核心参数 */
  --clay-shadow-offset: 3px;
  --clay-shadow-blur: 8px;
  --clay-shadow-color: rgba(107, 78, 61, 0.12);
  --clay-highlight-color: rgba(255, 255, 255, 0.7);
  --clay-bg-gradient: linear-gradient(135deg,
    rgba(255,255,255,0.4) 0%,
    rgba(255,255,255,0.1) 100%);
}
```

**Claymorphism 应用规则**：
- 所有可交互的凸起元素（按钮、卡片、字母牌）使用 Claymorphism 阴影
- 凹陷元素（输入框、棋盘空格）使用内阴影
- 软3D质感通过 `box-shadow` 双层阴影实现：底层为暖棕色投影，顶层为白色高光

#### 2.3.5 阴影系统

```css

```

## 3. 交互设计规范

### 3.1 导航系统

#### 3.1.1 底部 Tab 导航

应用主导航使用底部固定 Tab Bar，三个入口：

| Tab | 图标 | 路由 | 说明 |
|-----|------|------|------|
| 大厅 | 房子图标 | `/` | 游戏入口页，默认首页 |
| 单词本 | 书本图标 | `/wordbook` | 收藏单词列表 |
| 帮助 | 问号图标 | `/help` | Scrabble 规则说明 |

**Tab Bar 规格**：
- 高度：56px（含底部安全区则为 56px + env(safe-area-inset-bottom)）
- 背景色：`#FDF6E3`（Neutral-50） + 顶部 0.5px `#EDD9B5`（Neutral-200）分割线
- 图标大小：24px × 24px
- 文字大小：10px（--text-tiny）
- 选中态：图标和文字变为 `#FFD23F`（Primary-400），图标填充
- 未选中态：图标和文字为 `#6B4E3D`（Neutral-700），图标线条
- 每个 Tab 点击区域至少 44px × 44px（可访问性要求）
- 固定在视口底部，随页面内容滚动不消失

#### 3.1.2 页面跳转方式

| 跳转类型 | 动画 | duration | easing | 使用场景 |
|---------|------|---------|--------|---------|
| Tab 切换 | 无动画（即时替换） | 0ms | — | 底部导航切换 |
| 前进（push） | 从右向左滑入 | 300ms | `cubic-bezier(0.4, 0, 0.2, 1)` | 大厅→游戏、大厅→帮助 |
| 返回（pop） | 从左向右滑出 | 300ms | `cubic-bezier(0.4, 0, 0.2, 1)` | 游戏→大厅、帮助→大厅 |
| 浮层弹出（modal） | 从底部向上滑入 + 遮罩渐显 | 350ms | `cubic-bezier(0.32, 0.72, 0, 1)` | 单词详情、结算面板 |

### 8.4 CSS 架构建议

**推荐架构**：CSS Modules + CSS Custom Properties

```
src/
├── styles/
│   ├── tokens.css          ← CSS变量定义（Design Tokens）
│   ├── reset.css           ← 全局重置 + 基础样式
│   ├── base.css            ← 排版、链接、焦点等基础样式
│   ├── utilities.css       ← 工具类（间距、文字、flex等）
│   └── animations.css      ← 全局动画关键帧定义
├── components/
│   ├── Button/
│   │   ├── Button.tsx
│   │   └── Button.module.css
│   ├── TileLetter/
│   │   ├── TileLetter.tsx
│   │   └── TileLetter.module.css
│   ├── BoardCell/
│   │   ├── BoardCell.tsx
│   │   └── BoardCell.module.css
│   ├── TabBar/
│   │   ├── TabBar.tsx
│   │   └── TabBar.module.css
│   └── ... (其他组件)
└── pages/
    ├── Home/
    │   ├── Home.tsx
    │   └── Home.module.css
    ├── Game/
    │   ├── Game.tsx
    │   └── Game.module.css
    ├── WordBook/
    │   ├── WordBook.tsx
    │   └── WordBook.module.css
    └── Help/
        ├── Help.tsx
        └── Help.module.css
```

**CSS 编写原则**：
1. Mobile-first：基础样式为移动端，通过 `min-width` 媒体查询渐进增强
2. 所有颜色、间距、圆角、阴影使用 CSS 变量，不硬编码
3. 组件样式通过 CSS Modules 隔离，避免全局污染
4. 避免深层嵌套（最多 3 层）
5. 动画统一使用 `tokens.css` 中的过渡变量
6. Canvas 棋盘渲染样式通过 JavaScript 配置对象管理，不使用 CSS

### 8.6 性能优化建议

1. **字体加载**：使用 `font-display: swap` 确保文字立即可见
2. **SVG 图标**：合并为 sprite，减少 HTTP 请求
3. **Canvas 优化**：避免在每帧重绘整个棋盘，使用脏矩形（dirty rectangle）局部更新
4. **CSS 动画**：仅对 `transform` 和 `opacity` 做动画，避免触发 layout/paint
5. **图片**：Ollie SVG 内联在 HTML 中，减少请求
6. **关键 CSS**：首屏所需的核心样式内联在 `<head>` 中
7. **骨架屏**：使用纯 CSS（渐变 + 动画），不依赖 JS
